```bash

filebase upload ./assets

# 4. Integrate with IPFS
ipfs add -r ./game-assets
```

## 📋 TECHNOLOGY DECISION LOG

### Decisions Made

1. **Backend Framework**: .NET 8+ ✅
   - **Reason**: Mature ecosystem, excellent blockchain support via Nethereum, team expertise
   - **Alternative Considered**: Node.js with Web3.js
   - **Trade-off**: Slightly smaller Web3 community vs. .NET's enterprise readiness

2. **Smart Contract Development**: Hardhat + Foundry ✅
   - **Reason**: Best-in-class tooling, professional-grade debugging
   - **Alternative Considered**: Remix IDE
   - **Trade-off**: Steeper learning curve vs. faster development cycle

3. **Decentralized Storage**: Filebase ✅
   - **Reason**: S3-compatible, easy migration, geo-redundant
   - **Alternative Considered**: Filecoin direct
   - **Trade-off**: Slightly higher cost vs. significantly easier implementation

4. **Gaming Engine**: Unity + ChainSafe ✅
   - **Reason**: Industry standard, robust Web3 SDK, C# alignment
   - **Alternative Considered**: Unreal Engine, Xaya
   - **Trade-off**: Licensing costs vs. proven ecosystem

5. **Node Infrastructure**: QuickNode ✅
   - **Reason**: Best free tier, excellent for trading bots, low latency
   - **Alternative Considered**: Alchemy, self-hosted nodes
   - **Trade-off**: Vendor lock-in vs. operational simplicity

## 🎓 GLOSSARY

**Terms for VAMGUARD Team**:

- **3-Tier Architecture**: Presentation layer (UI) + Business logic layer (API) + Data layer (Database)
- **dApp**: Decentralized Application - runs on blockchain instead of central servers
- **EVM**: Ethereum Virtual Machine - the runtime environment for smart contracts
- **Gas**: Transaction fees paid to execute operations on blockchain
- **L2/Rollup**: Layer 2 scaling solution - processes transactions off main chain for speed
- **Multi-sig**: Multi-signature wallet requiring multiple approvals for transactions
- **NFT**: Non-Fungible Token - unique digital asset
- **RPC**: Remote Procedure Call - how apps communicate with blockchain nodes
- **Smart Contract**: Self-executing code deployed on blockchain
- **Testnet**: Test blockchain network for development (no real money)
- **Wallet**: Software to store and manage cryptographic keys
